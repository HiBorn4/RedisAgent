// How it works

Agent-driven, not hardcoded

The LLM drives every tool call. No query language, no fixed routes — just reasoning over your live Redis instance.

01

You ask a question

Type any question about your Redis data in plain English. The Streamlit UI sends it to the FastAPI backend.

POST /chat
02

ADK invokes the agent

Google ADK's LlmAgent receives the query with your custom system prompt and decides which MCP tools to call.

LlmAgent + Runner
03

MCP tools hit Redis

The agent chains list_keys → search_keys → get_key autonomously until it finds the answer.

MCP stdio
04

Answer surfaces in chat

The agent parses nested JSON inside Redis values and returns a clean, human-readable answer. Session state persists across turns.

SQLite session store
// Features

Everything you'd expect,
nothing you don't need

Built lean — every component earns its place.

🗣️

Conversational Redis access

Plain-English queries map to structured Redis lookups automatically. No query syntax to learn.

🔁

Persistent sessions

ADK's DatabaseSessionService keeps conversation memory across requests via SQLite.

🔌

MCP stdio transport

Redis tools are exposed as proper MCP primitives — plug-and-play with any MCP-compatible client.

🔍

Autonomous key discovery

Agent starts blind with list_keys(*) and narrows down using pattern search — no hardcoded key names.

🧩

Nested JSON reasoning

Values that are JSON strings get automatically parsed and drilled into. The agent finds the exact field requested.

⚡

FastAPI async backend

Fully async from HTTP layer to Redis calls. Production-ready with health endpoint and structured logging.

📦

Redis bulk data loader

redis_dump.py loads entire directories of JSON files into Redis with one command.

🐳

Docker ready

docker-compose up brings up Redis + backend + frontend in one shot, no manual setup needed.

// Quick Start

Up in under 5 minutes

One script does it all. Or go step-by-step below.

1
Clone and configure
git clone https://github.com/YOUR_USERNAME/redis-mcp-agent.git cd redis-mcp-agent cp .env.example .env # Edit .env — set REDIS_URL and GOOGLE_API_KEY
2
One-command start (recommended)
./start.sh

Or use Docker: docker-compose up

3
Open the chat UI
http://localhost:8501

Backend health: http://localhost:8001/health

4
Load data and start querying
# Drop JSON files into backend/redis_dumps/ python backend/redis_dump.py # Then ask the agent: # "List all available Redis keys" # "What is the trip status?" # "Fetch the travel itinerary data"
